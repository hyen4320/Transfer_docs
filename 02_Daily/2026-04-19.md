### BE: 25/26 이적시장 데이터 조건 검색 기능 개발

**현황:** `NewsController`는 `status`, `leagueId`만 지원. `season`, `window` 쿼리 파라미터 없음.

**검색 가능 요소 전체:**

| 파라미터 | 타입 | 값 | 상태 |
|---------|------|----|------|
| `status` | String | `INTEREST` / `RUMOR` / `CONFIRMED` / `DENIED` / `LOAN` / `CONTRACT_EXTENSION` | 지원 중 |
| `leagueId` | Long | 리그 ID | 지원 중 |
| `season` | Short | `49` (24/25), `51` (25/26) | **미지원** |
| `window` | String | `SUMMER` / `WINTER` | **미지원** |
| `journalistId` | Long | 기자 ID | **미지원** |
| `position` | String | `GK` / `DF` / `MF` / `FW` | **미지원** |
| `minFeeEur` | Long | 이적료 하한 (유로) | **미지원** |
| `maxFeeEur` | Long | 이적료 상한 (유로) | **미지원** |
| `sort` | String | `feeEur,desc` 등 | **미지원** (Top N 이적료 정렬용) |
| `page` / `size` | int | 기본값 0 / 20 | 지원 중 |

**예시 쿼리:**
```
GET /api/news?season=51&window=SUMMER&status=CONFIRMED&leagueId=1
GET /api/news?position=FW&minFeeEur=50000000&sort=feeEur,desc&size=100
GET /api/news?journalistId=1&season=51
```

**구현 완료:**
- `TransferNewsSearchCondition` (record) — 조건 파라미터 모음
- `TransferNewsSpecification` — JPA Criteria API 동적 쿼리
- `TransferNewsRepository` — `JpaSpecificationExecutor` 추가
- `TransferNewsService.search()` + `TransferNewsServiceImpl` 구현
- `NewsController` — 기존 if-else 제거, 단일 `search` 엔드포인트로 통합
- `TransferNewsSearchTest` — 필터별 통합 테스트 18개
- `NewsControllerTest` — 업데이트

### TEST: 유럽 5대리그 25/26 여름/겨울 이적시장 데이터 수집 후 적용

**현황:** `TransferNews`에 `season`, `window` 필드는 존재하나 자동 설정 로직 없음. Post → TransferNews 변환 파이프라인 자체가 수동.

**해결 방안 → 구현 완료** (`DB/TransferNews/crawl.py`)
- Transfermarkt HTML 크롤링으로 각 클럽의 Zugänge(Arrivals) 파싱
- 이적 날짜 기준 window 자동 판정 (6~9월 → SUMMER, 1~2월 → WINTER)
- `transfermarkt_bot` 더미 기자 + 합성 post 생성으로 `post_id NOT NULL` 제약 해결
- `to_club` DB 미존재 시 INSERT 자동 스킵 (5대 리그 외 구단 방어)
- season=51 고정, 출력: `transfer_news_2526.sql`

**실행 방법:**
```bash
cd D:\Transfer\DB\TransferNews
python crawl.py
# 출력: transfer_news_2526.sql
psql -U postgres -d transfer -f transfer_news_2526.sql
```
