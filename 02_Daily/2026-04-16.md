### 지도 마커 겹침 문제 해결 방안

D3 지도 위에 클럽 마커가 겹치는 경우 (예: 런던 구단들, 밀라노 Inter/AC) 대응 방법 정리.

#### 현재 구현: 반복 Push-apart (resolveOverlaps)
```ts
function resolveOverlaps(positions, minDist = 9) {
  // 20 iteration 한도로 겹치는 쌍을 서로 밀어냄
  // O(n²) × 20 — 구단 수(<50) 수준에선 충분히 빠름
}
```
- 장점: 구현 단순, 구단 수가 적을 때 성능 OK
- 단점: 원래 좌표에서 벗어날 수 있음, 많이 겹치면 수렴 불안정

#### 개선 옵션 (필요 시 검토)

**A. D3 Force Simulation**
```ts
import { forceSimulation, forceCollide, forceX, forceY } from 'd3-force';
forceSimulation(nodes)
  .force('collide', forceCollide(minDist))
  .force('x', forceX(d => d.origX).strength(0.3))
  .force('y', forceY(d => d.origY).strength(0.3))
  .stop().tick(120);
```
- 장점: 물리 기반으로 자연스럽게 분산, 원래 위치로 당기는 힘 조절 가능
- 단점: d3-force 의존 추가, tick 수만큼 CPU 소모

**B. 줌 레벨별 클러스터링**
- D3 zoom transform 값에 따라 가까운 마커를 하나의 클러스터 원으로 묶음
- 줌인 시 개별 마커 분리 표시
- 장점: 유저 경험 자연스러움, 마커 수 많아도 깔끔
- 단점: 구현 복잡도 높음 (클러스터 로직 별도 작성)

**C. Offscreen Label + 선 연결**
- 겹치는 마커를 지도 바깥 레전드 영역에 표시하고 선으로 연결
- 장점: 지도 공간 보존
- 단점: 마커가 2~3개 이상 겹칠 때 선이 복잡해짐

> 현재는 A(push-apart)로 충분. 구단 추가로 겹침이 심해지면 B(D3 force) 전환 권장.
