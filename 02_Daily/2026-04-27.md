### Priority 1 — 성능 최적화

#### fetch AbortController 추가
- 현재 `PlayerPanel.tsx`, `useNewsInfinite.ts` 등 모든 fetch에 AbortController 없음
- 빠르게 탭을 전환하거나 season을 바꾸면 stale 응답이 state를 덮어씀
- `useEffect` cleanup에서 `controller.abort()` 처리 필요
- [ ] `PlayerPanel`: playerId 바뀔 때 이전 요청 abort
- [ ] `useNewsInfinite`: season 바뀔 때 이전 페이지 요청 abort

#### handleFlyTo useCallback 누락
- `App.tsx`의 `handleFlyTo`가 일반 함수 — `useCallback` 빠져있음
- `handlePlayerClick`이 `handleFlyTo`를 deps로 쓰는데 매 렌더마다 새 참조 → 불필요한 재생성
- [ ] `handleFlyTo`를 `useCallback`으로 감싸기

#### WorldMap D3 atlas 캐싱 확인
- `loadWorldAtlas()` → `worldAtlas.ts`가 모듈 레벨에서 캐싱하는지 확인
- clubs/news 변경마다 `draw` useCallback이 재실행되는데 atlas fetch가 중복으로 나가면 안 됨
- [ ] `worldAtlas.ts` 내부 promise 캐싱 여부 확인, 없으면 module-level singleton 추가

#### React.memo 적용
- `NewsCard` — 부모(SidePanel) 리렌더 시 매번 전체 목록 리렌더
- `WorldMap` — clubs/news 외 변경에도 리렌더될 수 있음
- [ ] `NewsCard`에 `React.memo` 적용
- [ ] `WorldMap`은 props가 많아 shallowEqual 비교 비용 측정 후 결정

### Priority 2 — UX 개선

#### PlayerPanel 모바일 미지원
- 현재 `w-[460px]` 고정 — 모바일에서 화면 넘침
- SidePanel처럼 모바일에서 bottom sheet, 데스크탑에서 좌측 슬라이드로 분기 필요
- [ ] `useIsMobile()` 훅 적용 (SidePanel에서 로직 재사용)
- [ ] 모바일: `fixed bottom-0 left-0 right-0 h-[80vh]` + 드래그 핸들

#### handlePlayerClick 실패 시 피드백 없음
- `fetchPlayersSearch` 결과가 0건이면 조용히 종료 — 사용자 입장에서 반응 없음
- [ ] 결과 없을 때 토스트 또는 SidePanel에 "Player not found" 표시

#### CountryMapPage ← PlayerPanel 연동
- `CountryMapPage`가 열려있을 때 PlayerPanel을 열면 `leftOffset`이 갱신 안 됨
- `App.tsx`에서 `playerPanelId != null` 조건이 `CountryMapPage leftOffset`에 반영은 됐으나
  CountryMapPage가 이미 마운트된 상태에서 prop이 변하는지 확인 필요
- [ ] CountryMapPage `leftOffset` prop 변화 시 지도 리사이즈 동작 확인

#### Footer 링크 세 개가 모두 `/info`
- About, Contact, Privacy 셋 다 `/info`로 연결됨 — 앵커(#section)로 분기하거나 별도 라우트 정리
- [ ] `/info` 내부 섹션별 anchor id 추가 후 `href="/info#privacy"` 식으로 연결

#### 시즌 셀렉터 중복
- `App.tsx` floating bottom-left 셀렉터와 `SidePanel` 내부 셀렉터 두 곳에 동일한 옵션 배열 생성
- `MAP_SEASON_OPTIONS` (App) / `SEASON_OPTIONS` (SidePanel) — 같은 로직 중복
- [ ] `src/data/constants.ts`로 분리 후 공유
