## Trending BE 구현 메모

### 엔드포인트
`GET /api/news/trending-players?days=7&limit=5`

### 쿼리
```sql
SELECT n.player, COUNT(n.id) as cnt
FROM news n
WHERE n.published_at > NOW() - INTERVAL ':days days'
  AND n.player IS NOT NULL AND n.player != ''
GROUP BY n.player
ORDER BY cnt DESC
LIMIT :limit
```

### Spring 구현 포인트
- `@Cacheable` (Caffeine, TTL 1시간) — 매 요청마다 GROUP BY 집계 방지
- `@Scheduled(fixedRate = 3_600_000)` + `@CacheEvict` 로 주기적 갱신
- FE: `fetchLeagues()` 패턴처럼 마운트 시 호출 → `TRENDING` 상수 대체
